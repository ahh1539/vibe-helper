# AGENTS.md

## Version Control

NEVER EVER PUBLISH ANY BRANCH IN THIS REPOSITORY!

All branches in this repository are for development purposes only and should never be published or made publicly accessible. This includes but is not limited to:
- Pushing to remote repositories
- Creating pull requests without explicit permission
- Sharing branch contents externally
- Merging to main/master without proper review

## Git Workflow

1. Always create feature branches from main
2. Keep branches up to date with main through regular rebasing
3. Never force push to shared branches
4. Get explicit approval before merging any changes
5. Follow semantic commit messages

## Branch Naming Convention

Use descriptive branch names following this pattern:
- `feat/feature-name` for new features
- `fix/bug-description` for bug fixes
- `docs/documentation-updates` for documentation changes
- `refactor/code-improvements` for code refactoring
- `chore/maintenance-tasks` for maintenance tasks
